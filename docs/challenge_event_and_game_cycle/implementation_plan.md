# 朝のランダムイベント機能 実装計画

営業開始前（1日の始まり）に、様々な効果をもたらす「朝のランダムイベント」を発生させるシステムを実装します。これにより、単調になりがちな日々のプレイにスパイスを加えます。

## 変更内容 (Proposed Changes)

### 1. イベントデータの定義 (`src/lib/events.ts` を新規作成)
#### [NEW] `src/lib/events.ts`
発生する可能性のあるイベントのリストを定義します。
*   **インフルエンサーの来店** (Good): 一時的にフォロワーが急増する。
*   **悪天候（台風など）** (Bad): その日1日の来客数（Traffic）が減少する。（または固定で資金がマイナスになる等）
*   **冷蔵庫の故障** (Bad): ショーケースに出しているメニューの「鮮度」が一気に1日分落ちる。
*   **謎のスポンサー** (Good): 開発支援金として臨時収入（例: +10,000円）がもらえる。
※確率は毎朝 30%〜40% 程度で発生するようにします。

### 2. 状態管理の拡張 (`src/store/gameStore.ts`)
#### [MODIFY] `src/store/gameStore.ts`
*   `GameState` に `pendingMorningEvent: string | null;` （発生待ちのイベントID）を追加します。
*   `advanceDay` （1日の終了時）の処理で、確率でランダムにイベントIDを選び `pendingMorningEvent` にセットします。
*   イベントの効果を適用し、ポップアップを閉じるためのアクション `resolveMorningEvent()` を追加します。

### 3. イベント表示用UIの作成 (`src/components/game/MorningEventDialog.tsx`)
#### [NEW] `src/components/game/MorningEventDialog.tsx`
*   `pendingMorningEvent` が存在する場合に画面の最前面に表示されるダイアログ（ポップアップ）コンポーネントを作成します。
*   イベントのタイトル、説明文、および結果を表示し、「OK」ボタンを押すことで効果が適用されて1日がスタートする流れになります。

#### [MODIFY] `src/app/page.tsx`
*   作成した `MorningEventDialog` をページ全体で読み込むように追加し、どのタブを開いていても朝一番にイベントが表示されるようにします。

## Verification Plan

*   1日進めた際、一定確率で朝にイベントダイアログが表示されるか。
*   「臨時収入」なら資金が増える、「冷蔵庫故障」ならショーケースのメニューの鮮度が落ちるなど、イベント固有の効果が正しく状態に反映されるか。
*   ダイアログを閉じた後は、通常通りゲームを進行できるか。
