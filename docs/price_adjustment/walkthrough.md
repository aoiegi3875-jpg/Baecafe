# 実装完了レポート：販売価格の調整機能

## 実装内容
メニューをショーケースに並べる際に、AIが決定した「相場（推奨価格）」を基準にして、プレイヤーが自由に販売価格を調整できる機能を実装しました。
また、設定した価格に応じて売れ行きや炎上リスクが変化する経営シミュレーションとしてのロジックも追加しています。

### 1. 価格設定ダイアログの追加
`Showcase.tsx` において、ギャラリーから「並べる」ボタンを押した際、即座に並べるのではなく**「販売価格の設定ダイアログ」**が表示されるようにしました。
- スライダー（シークバー）を使って、相場価格の **50%（半額）〜 200%（2倍）** の範囲で 10円単位で価格を調整できます。
- 価格を操作すると、ダイアログ下部のメッセージが動的に変化します。
  - **安い場合**: 「✨ お買い得: 飛ぶように売れてフォロワー増！」という緑色の通知が出ます。
  - **高い場合**: 「⚠️ ぼったくり注意: 炎上リスクが高まります！」という赤色の警告が出ます。

### 2. ショーケースUIの変更
設定した独自の価格（`customPrice`）がショーケースのカードに大きく表示されるようになりました。
また、相場から価格を変更した場合は、元の相場価格が取り消し線付きで小さく併記されるため、どれくらい調整したかが一目でわかります。

### 3. 営業ロジックへの影響（ゲームバランス）
価格の変更幅（価格比率 = 設定価格 / 相場価格）に基づいて、以下の影響が出るように `handleOperate` を改修しました。
- **販売数**: 価格が安いほど売れる数が増え、高いほど売れにくくなります。
- **フォロワー増減**: お買い得な価格（相場より安い）であるほど、お店の評判が良くなりフォロワー獲得数にボーナスがつきます。
- **炎上ペナルティ**: 相場の1.5倍以上の「ぼったくり価格」を設定して営業すると、その日の炎上リスクゲージに大きなペナルティ（+30など）が加算され、炎上イベントが非常に起きやすくなります。

## 確認事項
ブラウザ（http://localhost:3000）から以下の手順で動作をご確認ください。
1. 「営業」タブに移動し、下部のギャラリーから適当なメニューの「並べる」ボタンを押す。
2. 価格設定ダイアログが開くので、スライダーを動かして価格を極端に安くしたり、高くしたりしてメッセージの変化を確認する。
3. 価格を決定してショーケースに並べ、カードの価格表記（相場と設定価格）を確認する。
4. 「1日営業する」を押し、設定した価格による売上やフォロワーの増え方、炎上ゲージの動きを確認する（特にぼったくり価格で炎上するかどうか）。
