# 価格調整機能の実装計画

メニューをショーケースに並べる際に、AIが判定した「相場価格」を基準にして、実際の「販売価格」をプレイヤーが調整できる機能を実装します。
また、設定した価格によって売上や評判（フォロワー・炎上リスク）に影響が出るようなゲームバランスのロジックを追加します。

## Proposed Changes

### 1. データ構造の変更 (`src/store/gameStore.ts`)
- `ActiveMenu`（ショーケースに並んでいるメニュー）の型に `customPrice: number` を追加します。
- `addToShowcase` アクションの引数を `(menu: Menu, customPrice: number)` に変更します。

### 2. ショーケース UI の変更 (`src/components/game/Showcase.tsx`)
- ギャラリーのリストから「並べる」ボタンを押した際、即座に並べるのではなく**「価格設定ダイアログ」**を開くようにします。
- ダイアログでは、スライダー（Slider）または数値入力を用いて、相場価格の **-50% 〜 +100%** 程度の幅で販売価格を調整できるようにします。
- 調整した価格によって、「売れ行き予測」のテキストが変化するような簡易的なプレビューを表示します（例：「お買い得！売れ行きアップ！」「ぼったくり価格...炎上するかも？」など）。
- ショーケースのカード表示で、相場価格と設定価格の両方、または設定価格メインの表示に変更します。

### 3. 販売ロジック（営業時の処理）の変更 (`src/components/game/Showcase.tsx` の `handleOperate`)
設定した `customPrice` と相場価格 `menu.price` の比率（`priceRatio = customPrice / menu.price`）を計算し、以下の影響を与えます。

- **販売数 (unitsSold) への影響**:
  価格が安いほど売れやすく、高いほど売れにくくします。
  `unitsSold` の計算式に `(2.0 - priceRatio)` のような係数を掛けます（価格が1.5倍なら売れ行き半減、0.5倍なら1.5倍売れる）。
- **フォロワー増加への影響**:
  安いとお得感でフォロワーが増えやすく、高いと増えにくくします。
- **炎上リスクへの影響**:
  相場より大幅に高い（例: 1.5倍以上）場合、ボッタクリとしてその日の `maxRisk`（炎上リスク）にペナルティ（+20〜30など）を加算し、炎上イベントの発生確率を上げます。
