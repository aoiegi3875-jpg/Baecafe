# バックグラウンドメニュー開発の実装計画

メニュー開発中（AI生成中）に別タブに移動しても動作が継続し、かつ全画面でローディング状態がわかるように改修します。

## 課題
現在、メニュー生成中のローディング状態 (`isGenerating`) や、生成された直後の結果 (`result`) は `MenuDevelopment.tsx` コンポーネント内のローカルステートで管理されています。
そのため、別タブに移動してコンポーネントがアンマウントされると、画面上のローディング表示が消え、結果も表示されなくなってしまいます（裏側の処理自体は継続し、ギャラリーには追加されますが、UXとして不親切です）。

## 解決策
ローディング状態と最新の生成結果を、グローバルな状態管理（Zustand の `gameStore.ts`）に移動させます。

### 1. `gameStore.ts` の改修
以下のステートとアクションを追加します。
- `isGeneratingMenu: boolean` （生成中かどうかのフラグ）
- `latestGeneratedMenu: Menu | null` （最新の生成結果）
- それぞれの setter 関数

※ページの更新（リロード）時に生成中ステートが残り続けるのを防ぐため、アプリの初期化時（`page.tsx` マウント時）に `isGeneratingMenu` を `false` にリセットする処理も入れます。

### 2. `TopBar.tsx` （常に表示されるヘッダー）の改修
`isGeneratingMenu` が `true` の場合、ヘッダーに「👨‍🍳 AIシェフが新メニューを考案中...」といったアニメーション付きのローディングバッジを表示します。これにより、どのタブ（営業やショップなど）にいても、裏でAIが働いていることがわかります。

### 3. `MenuDevelopment.tsx` の改修
ローカルステートであった `isGenerating` と `result` を廃止し、`gameStore` の `isGeneratingMenu` と `latestGeneratedMenu` を参照するように書き換えます。
これにより、別タブに行ってから戻ってきても、生成が終わっていれば結果のカードが表示されたままになります。

## User Review Required

「別タブにいてもヘッダーで生成中であることがわかる」「戻ってきたらちゃんと生成結果が表示されている」という挙動になります。
